# Implementing a Reusable Blog Template System

This guide outlines the architectural steps and instructions required to build a scalable, reusable blog template system. It covers the creation of a List View, Detail View, Breadcrumb navigation, and a Media Viewer for Google Drive assets.

## 1. Data Architecture & Storage

### Current Approach: Google Drive Integration
Currently, your architecture relies on fetching assets (images/videos) and content (a README file) directly from Google Drive.
* **Media:** Google Drive files must be set to "Anyone with the link can view". To embed these files, the standard shareable link must be modified from the `view` format to a direct export format.
* **Content:** Reading a Markdown/README file from Google Drive requires calling the Google Drive API, downloading the file content, and parsing it on the client or server side.

### 💡 Recommended Implementation (Better Alternatives)
While Google Drive works for personal storage, it is not optimized for web content delivery (no built-in caching, API rate limits, complex markdown parsing). Consider these alternatives:
1.  **Headless CMS (Recommended):** Use tools like Sanity, Contentful, or Strapi. They provide a user-friendly editor, manage media delivery via CDN, and return structured JSON that is incredibly easy to map to your views.
2.  **Git-Backed Markdown (MDX):** Store `.md` or `.mdx` files directly in your repository. This is extremely fast, requires no external API calls, and supports embedding custom components directly in the text.
3.  **Notion API:** If you prefer a document-style writing experience, you can use Notion as your CMS and fetch the content via the official Notion API.

## 2. Component Implementation Instructions

*(Note: Ensure all components utilize your global design system, CSS variables, or shared utility classes to maintain 100% UI consistency with the rest of the website.)*

### A. Breadcrumb Navigation Component
**Purpose:** Help users understand their current location in the blog hierarchy and easily navigate back.
**Instructions:**
1.  Create a component that reads the current URL path.
2.  Split the path into readable segments (e.g., `Home > Blog > Current Article`).
3.  Map over the segments to create clickable links for all items except the last one (which should be plain text representing the current page).
4.  **UI Consistency:** Use the standard typography for secondary links on your website. Add a subtle hover effect that matches your global button/link states. Use a standard separator icon (like a chevron or slash) from your existing icon library.

### B. Blog List View Component
**Purpose:** Display a grid or list of available blog articles.
**Instructions:**
1.  Fetch the list of available blogs from your data source.
2.  Map through the data to render "Blog Cards". Each card should contain:
    * A thumbnail image (fetched from the Google Drive link).
    * The blog title.
    * A short excerpt or summary.
    * Publish date and author (if available).
3.  Implement pagination or a "Load More" mechanism if you expect a large number of posts.
4.  **UI Consistency:** Apply the exact same card shadow, border-radius, and padding logic used elsewhere on the site. Ensure the grid is fully responsive, collapsing from 3 columns on desktop to 1 column on mobile.

### C. Blog Detail View Component
**Purpose:** Render the full content of the selected blog post.
**Instructions:**
1.  Fetch the specific README/Markdown content for the requested post.
2.  Pass the raw text into a Markdown parser library.
3.  Map standard Markdown elements (h1, h2, p, ul, blockquote) to your website's custom typography components. This ensures a standard `<h1>` in the markdown looks exactly like the headers on your homepage.
4.  Structure the layout to include a generous reading width (e.g., max-width 700px - 800px) centered on the screen to optimize readability.
5.  Include the Breadcrumb component at the very top of this view.

### D. Image Viewer & Video Player (Google Drive Media)
**Purpose:** Display media embedded within the blog post smoothly.
**Instructions:**
1.  Create a custom wrapper component for media.
2.  **For Images:** Intercept image tags generated by your Markdown parser. Extract the Google Drive ID and construct a direct image source link. Wrap the image in a container that enforces a consistent aspect ratio and handles loading states (e.g., showing a skeleton loader until the image resolves).
3.  **For Videos:** Intercept video links. Use a standard HTML5 `<video>` tag or a library wrapper. Provide the direct Google Drive download link as the source. Include standard controls (play, pause, volume, fullscreen).
4.  **UI Consistency:** Apply standard border-radius to media containers. If implementing a lightbox (click to expand), ensure the modal overlay matches the site's existing modal styling (background opacity, close button placement, animation).
